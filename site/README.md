# AI 한 숟가락 — 브리핑 사이트

빌드 도구 없이 순수 HTML/CSS/JS로만 만든 정적 사이트입니다.
GitHub 저장소에 그대로 올리고 GitHub Pages만 켜면 바로 공개됩니다.

## 폴더 구조

```
site/
├── index.html          ← 메인 페이지
├── admin.html          ← 브라우저에서 바로 발행하는 페이지
├── assets/
│   ├── style.css        ← 디자인
│   ├── app.js            ← 목록을 화면에 그려주는 스크립트
│   ├── posts.js          ← ★ 매일 수정될 파일 (글 목록 데이터)
│   └── thumbs/            ← 카드 썸네일 이미지
└── pdfs/
    ├── 2026-07-21.pdf
    └── 2026-07-23.pdf
```

## GitHub Pages로 배포하기 (최초 1회)

1. github.com 에서 새 저장소(Repository)를 만듭니다. (예: `briefing`)
2. 이 `site` 폴더 안의 내용물(파일들)을 그 저장소에 그대로 업로드합니다.
   (저장소 페이지의 "Add file → Upload files"로 드래그 앤 드롭하면 됩니다.)
3. 저장소의 **Settings → Pages** 로 들어갑니다.
4. "Branch" 항목에서 `main` 브랜치, 폴더는 `/ (root)` 를 선택하고 저장합니다.
5. 1~2분 뒤 `https://[깃허브아이디].github.io/briefing/` 주소로 접속하면 사이트가 보입니다.

## 매일 브리핑 올리는 방법 (구글 드라이브 사용)

GitHub 저장소에는 PDF를 직접 올리지 않고, 구글 드라이브에 올린 뒤 링크만 연결하는 방식입니다.
저장소 용량이 늘지 않아서 개수 제한 걱정 없이 계속 쓸 수 있습니다.

1. 구글 드라이브에 브리핑 전용 폴더를 하나 만들어 둡니다. (예: `브리핑 아카이브`)
2. 오늘 만든 PDF를 그 폴더에 업로드합니다.
3. 업로드한 파일을 **우클릭 → 공유** → 일반 액세스를 **"링크가 있는 모든 사용자"** + **"뷰어"**로
   설정하고 **링크 복사**를 누릅니다.
   (이 설정이 안 되어 있으면 방문자가 PDF를 못 엽니다. 꼭 확인하세요.)
4. `assets/posts.js` 파일을 엽니다 (GitHub에서 연필 아이콘을 누르면 바로 수정 가능).
5. `POSTS` 배열 맨 위에 아래 형식으로 한 칸을 추가하고, `pdf` 값에 방금 복사한 링크를 붙여넣습니다.

```js
{
  date: "2026-07-24",
  title: "오늘 브리핑 제목",
  subtitle: null,                    // 부제가 없으면 null
  summary: "두세 문장짜리 소개 글.",
  tags: ["일일 브리핑", "태그2", "태그3"],
  pdf: "https://drive.google.com/file/d/여기에_긴_ID/view?usp=sharing",
},
```

6. 변경 사항을 커밋(Commit changes)하면 1분 안에 사이트에 자동 반영됩니다.

> 기존에 `pdfs` 폴더에 넣어둔 파일이 있다면 그대로 둬도 됩니다.
> `pdf` 값에 로컬 경로(`pdfs/2026-07-21.pdf`)를 쓰든, 구글 드라이브 링크를 쓰든 둘 다 동작합니다.

## 브라우저에서 바로 발행하기 (admin.html)

GitHub 화면에 들어가지 않고, 사이트 안의 `admin.html`에서 폼만 채우면 자동으로
`assets/posts.js`에 커밋됩니다. 최초 1회만 설정하면 그다음부터는 폼 작성 → 발행 버튼으로 끝입니다.

### 최초 설정 (1회)

1. GitHub 우측 상단 프로필 → **Settings** → **Developer settings** →
   **Personal access tokens** → **Fine-grained tokens** → **Generate new token**
2. Repository access에서 **이 저장소 하나만** 선택 (Only select repositories)
3. Permissions → **Contents** → **Read and write**로 설정
4. 토큰 생성 후 복사 (한 번만 보여주므로 꼭 복사해두기)
5. 배포된 사이트에서 `/admin.html`로 접속 (또는 하단 "발행하기" 링크)
6. GitHub 사용자명, 저장소 이름, 브랜치(`main`), 방금 만든 토큰을 입력하고 **저장**
   (이 정보는 이 브라우저에만 저장되며 다른 곳으로 전송되지 않습니다)

### 매일 발행할 때

1. `/admin.html` 접속
2. 날짜·제목·부제·요약·태그·PDF 링크(구글 드라이브 공유 링크)를 입력
3. 표지 썸네일 이미지가 있으면 선택 (없어도 됨)
4. **발행하기** 클릭 → 1~2분 뒤 사이트에 반영

### 주의할 점

- 토큰은 **본인 기기·본인 브라우저에서만** 등록하세요. 공용 컴퓨터라면 사용 후
  "① 저장소 연결" 섹션의 **지우기** 버튼으로 반드시 삭제하세요.
- 토큰이 새어나가면 그 저장소에 대한 쓰기 권한이 노출됩니다. GitHub 설정 화면에서
  언제든 토큰을 삭제(폐기)할 수 있습니다.
- `admin.html`은 검색엔진에 노출해도 토큰 없이는 아무 동작도 하지 않지만,
  깔끔하게 관리하고 싶다면 `robots.txt`로 색인을 막아도 됩니다.

## Codex한테 다음에 시켜볼 만한 것

- "posts.js를 읽고 오늘 날짜 카드를 자동으로 추가하는 스크립트 만들어줘"
- "admin.html에 발행 기록(최근 발행한 글 목록) 보여주는 기능 추가해줘"
- "이메일 구독 기능을 붙이고 싶은데 Buttondown이나 Mailchimp 연동 코드 짜줘"

## 나중에 확장하고 싶다면

- **이메일 발송**: Buttondown, Mailchimp 등 뉴스레터 서비스를 붙이면 이 사이트를 그대로 두고
  구독자에게 이메일도 보낼 수 있습니다.
- **검색/태그 필터**: 글이 많아지면 `app.js`에 태그별 필터링을 추가할 수 있습니다.
- **책 원고 재활용**: `posts.js`의 summary들은 나중에 책 목차나 서문 자료로 바로 가져다 쓸 수 있습니다.
